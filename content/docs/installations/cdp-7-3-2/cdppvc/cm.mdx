With the repositories and parcels hosted locally on the `cldr-mngr` node, Cloudera Manager itself can be installed. Cloudera Manager is the end-to-end management application used to install, configure and operate Cloudera on premises Base - everything from here to the end of this chapter is driven from it.

Cloudera Manager needs a database before it needs anything else. It uses various databases and datastores to store information about its own configuration, as well as information such as the health of the system or task progress - so the external PostgreSQL server is built first, then the Cloudera Manager Server packages are installed against it.

| Step | Page |
|:--|:--|

## Before you start

The external PostgreSQL DB server is installed on the **`cldr-mngr`** host, alongside the local `httpd` repository server set up in the [preliminary work](/installations/cdp-7-3-2/post-os-work). The database and the Cloudera Manager Server therefore share a host in this reference build, which is why the certificate import in the next two pages is a local file copy rather than a transfer.

For the supported combinations of Cloudera Manager, Cloudera Runtime, Data Services, JDK, Python and PostgreSQL, check the compatibility matrix: [https://supportmatrix.cloudera.com/](https://supportmatrix.cloudera.com/)
