> **Optional - skip this step.** All four procedures on this page are marked *"Optional - Skip this Step"* in the source. Without them the services fall back to their existing authentication: Ranger to UNIX, Atlas to the local OS users on the host where the Atlas server runs.

This page configures **service** authentication against the directory. It is a different page from [Configure CM for external authentication using LDAP](/installations/cdp-7-3-2/cdppvc/cm/ldap-auth), which does the same job for Cloudera Manager itself. The directory is the FreeIPA server stood up in [Post OS Installation - Preliminary Work](/installations/cdp-7-3-2/post-os-work), and the bind password is the KDC admin password configured there.

Every screenshot on this page is recycled from an unrelated Active Directory lab - see [Version strings and lab domains in this section](#version-strings-and-lab-domains-in-this-section) at the foot of the page. **Use the tables, never the captures.**

## 1. Configure Ranger authentication for LDAP

Follow steps below to configure Ranger for LDAP authentication.

**Step 1.** In Cloudera Manager, select **Ranger**, then click the **Configuration** tab.

**Step 2.** To display the authentication settings, type `authentication` in the Search box. Scroll down to see all of the LDAP settings.

**Step 3.** Select LDAP for "Admin Authentication Method".

![Ranger Configuration tab searched for authentication, with Admin Authentication Method set to LDAP among the UNIX, LDAP, ACTIVE_DIRECTORY, PAM and NONE options, and Admin UNIX Auth Remote Login checked](/images/cdp-7-3-2/cdppvc/base/ldap-auth/ranger-admin-authentication-method.png)

**Step 4.** Configure the following settings for LDAP authentication as shown below, the details depends on your configuration, based on existing LDAP/AD setup:

| Admin LDAP/AD Auth Bind User/ Bind DN: | `uid=admin,cn=users,cn=accounts,dc=cldrsetup,dc=local` |

## 2. Configure Hue for LDAP Authentication

Configuring Hue for Lightweight Directory Access Protocol (LDAP) enables you to import users and groups from a directory service, synchronize group membership manually or automatically at login, and authenticate with an LDAP server. Hue supports Microsoft Active Directory (AD) and open standard LDAP such as OpenLDAP and Forgerock OpenDJ Directory Services.

**Step 1.** Login to Cloudera Manager. Go to **Cluster > Hue > Configuration**.

**Step 2.** Change value for Authentication Backend:

```text
desktop.auth.backend.LdapBeckend,desktop.auth.backend.AllowFirstUserDjangoBackend
```

![Hue Authentication Backend property, backend / auth_backend, set service-wide to desktop.auth.backend.LdapBackend,desktop.auth.backend.AllowFirstUserDjangoBackend](/images/cdp-7-3-2/cdppvc/base/ldap-auth/hue-authentication-backend.png)

The prose spells the first class `LdapBeckend` while the screenshot shows `LdapBackend`; the prose is transcribed as printed, and the screenshot spelling is the one Hue actually accepts.

**Step 3.** Edit value for LDAP configuration. Example values set are shown in the screenshots below:

![Hue LDAP configuration part 1 - LDAP URL, LDAP Server CA Certificate, Enable LDAP TLS checked, Active Directory Domain, LDAP Username Pattern, Use Search Bind Authentication checked, Create LDAP users on login checked and LDAP Search Base](/images/cdp-7-3-2/cdppvc/base/ldap-auth/hue-ldap-config-1.png)

![Hue LDAP configuration part 2 - LDAP Bind User Distinguished Name, Bind Password, Username and Group Name for Test LDAP Configuration, LDAP User Filter, Username Attribute, Group Filter, Group Name Attribute and Group Membership Attribute](/images/cdp-7-3-2/cdppvc/base/ldap-auth/hue-ldap-config-2.png)

**Step 2.** Edit LDAP configuration. Sample configuration is shown in the screenshots below:
