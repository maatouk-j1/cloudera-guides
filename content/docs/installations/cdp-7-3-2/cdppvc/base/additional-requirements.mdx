## 3. Disable the Kerberized Web UIs

**Step 2.** Disable Kerberized Web-UI for YARN, Spark, HBase, HDFS, etc. from configuration:

While accessing Web UIs from web browsers, for HDFS-Namenode, YARN services, HiveServer2, Impala Services, etc. If gets 401 unauthorized error: We need to enable non-kerberized webUI for those services, for this we need to:

- Login to CM-UI, go to individual services, go to the **Configuration** section for individual services, search for **Kerberos**.
- Disable the option, by unchecking the checkbox, for Kerberos authentication for WebUI. Save changes. Restart the Stale Services to update the backend configuration files.

**Example Screenshots for HDFS. You can follow the same for other services, which requires WebUI access.**

![Cloudera Manager Home page with the HDFS service outlined in red in the PvCBaseCluster1 service list, beside the cluster charts](/images/cdp-7-3-2/cdppvc/base/additional-requirements/cm-home-hdfs.png)

![HDFS Configuration tab filtered on the search term kerberos, with the Enable Kerberos Authentication for HTTP Web-Consoles checkbox outlined in red and marked Uncheck this, and Save Changes highlighted at the bottom right](/images/cdp-7-3-2/cdppvc/base/additional-requirements/hdfs-kerberos-webui-config.png)

Then work through the following:

**1.** For Impala, Hive, Hive on Tez edit value for `Ranger Plugin URL Auth Filesystem Schemes` - `file:,wasb:,adl:`

**2.** Disable ~~Enable~~ **Kerberos Authentication for HTTP Web-Consoles** - HBase (Service-Wide), YARN, Spark2, Spark3, HiveServer2, Impala, HDFS, etc. ~~Click on Generate missing credentials for Kerberos.~~

**3.** For TLS/SSL enabled HDFS configuration you might see a warning as *"DataNode configuration is valid, but not recommended. There are two recommended configurations: (1) DataNode Transceiver Port and Secure DataNode Web UI Port (TLS/SSL) both `>=` 1024, DataNode Data Transfer Protection set, Hadoop TLS/SSL enabled; (2) DataNode Transceiver Port and DataNode HTTP Web UI Port both `<` 1024, DataNode Data Transfer Protection not set, Hadoop TLS/SSL disabled."*

```text
DataNode Transceiver Port (dfs_datanode_port)- 9866
DataNode HTTP Web UI Port (dfs.datanode.http.address) - 9864
DataNode Data Transfer Protection (dfs.data.transfer.protection) – Authentication
```

**4.** Install Ranger Plugin for services, add service dependency i.e. Hive etc. and restart the cluster. Ensure that the Ranger Solr and Ranger HDFS plugins are enabled. See [Additional Steps for Apache Ranger](https://docs.cloudera.com/cdp-private-cloud-base/7.1.9/installation/topics/cdpdc-additional-steps-ranger.html) for more details on Configuration Steps.

**5.** Make Sure HDFS, Ozone services are Installed and Running Successfully (critical services – if not working properly, then no other service will work properly): 1. HDFS 2. Zookeeper 3. …….

**6.** Atlas and Knox work with PAM authentication i.e. Local (Non-LDAP) users created on the base-master node where your Atlas server is running, unless Atlas is explicitly configured (integrated) to use LDAP. So you may need to create a local user on the base-master node, if it does not already exist.

**7.** Atlas is having dependencies on some additional services i.e. HBase, SOLR and Kafka

> **Note on Step 32 of the previous page:** the DataNode ports the Configure Kerberos stage set were **1004** and **1006** - both below 1024, which is configuration (2) in the warning above. Item 3 here moves them to **9866** and **9864** and sets Data Transfer Protection to `Authentication`, which is configuration (1). Apply one or the other, not a mix.
