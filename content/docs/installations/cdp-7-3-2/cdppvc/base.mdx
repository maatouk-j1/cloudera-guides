In this step, we will setup the Base cluster which will serve as the DataLake for the CDP Data Services that need the SDK capabilities for the cluster wide features like lineage, governance, security etc..,

Cloudera Manager is now installed, secured with Auto-TLS and Kerberos, monitoring itself through the Cloudera Management Service and authenticating against the directory. Everything from here is driven from its web UI: the Base cluster is built by adding hosts, distributing parcels, then selecting and configuring the services that make up the Data Lake.

## Before you start

The Base cluster hosts are the ones prepared during the [preliminary work](/installations/cdp-7-3-2/post-os-work) - `pvcbase-master` and `pvcbase-worker[1-3]`, all joined to the FreeIPA domain and all reachable from the Ansible control node. The wizard installs the Cloudera Manager Agent onto each of them over SSH, so password-less login or the root password must be available.

The local `httpd` repository server on `cldr-mngr` must already be serving both the Cloudera Manager agent packages and the Runtime and Spark parcels - that is what the [Setup Repositories and Parcels](/installations/cdp-7-3-2/cdppvc/repos-and-parcels) page built. The wizard asks for those URLs by hand; nothing here reaches the public internet.
